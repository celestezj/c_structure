# BSD 队列/链表库 (queue.h) 使用说明
## 概述
本 `queue.h` 是**跨平台兼容的 BSD 经典内核级（侵入式）链表/队列宏库**，纯 C 宏实现，无任何依赖，可直接用于用户态程序、嵌入式、工具开发、教学学习。

库中提供 **4 种高性能链表结构**，全部支持 O(1) 级别的插入、删除、遍历，是 C 语言底层开发最常用的数据结构工具集。

---

## 实现的 4 种数据结构
### 1. SLIST — 单向链表（Singly-Linked List）
- **特点**：单向、无尾指针、仅支持头部快速操作
- **优点**：结构最简单、内存占用最小
- **缺点**：尾部插入慢、不支持反向遍历、中间删除需要遍历
- **适用场景**：简单单向遍历、栈结构、内存极度紧张的场景

### 2. STAILQ — 单向尾队列（Singly-Linked Tail Queue）
- **特点**：单向链表 + 尾指针，**头尾 O(1) 插入**
- **优点**：支持快速尾插，比 SLIST 更实用
- **缺点**：不支持反向遍历
- **适用场景**：FIFO 队列、消息队列、有序链表
- **扩展**：内置 `STAILQ_REMOVE_CHECK` 安全删除宏，防止删除不存在节点

### 3. LIST — 双向链表（Doubly-Linked List）
- **特点**：双向、无尾指针、**任意节点 O(1) 删除**
- **优点**：支持前后插入、任意节点快速删除
- **缺点**：无尾指针、不支持反向遍历、尾部操作慢
- **适用场景**：需要频繁任意位置删除、但不需要尾操作的场景

### 4. TAILQ — 双向尾队列（Tail Queue）
- **特点**：双向链表 + 头尾指针 + **反向遍历**，功能最强大
- **优点**：O(1) 头尾插删、任意节点前后插入、正向/反向遍历
- **缺点**：结构稍复杂，占用内存略多
- **适用场景**：内核队列、消息链表、通用业务链表、**工业级首选**

## 核心能力对比表
| 结构 | 方向 | 尾指针 | 反向遍历 | 头插 | 尾插 | 任意删 | 任意前后插 |
|------|------|--------|----------|------|------|--------|-------------|
| SLIST | 单向 | ❌ | ❌ | O(1) | O(n) | O(n) | ❌ |
| STAILQ | 单向 | ✅ | ❌ | O(1) | O(1) | O(n) | ❌ |
| LIST | 双向 | ❌ | ❌ | O(1) | O(n) | O(1) | ✅ |
| TAILQ | 双向 | ✅ | ✅ | O(1) | O(1) | O(1) | ✅ |

## 配套测试 Demo（完整可运行）
库配套提供 4 个标准测试文件，覆盖**所有核心 API**：

1. `demo_slist.c` — SLIST 单向链表完整测试
2. `demo_stailq.c` — STAILQ 单向尾队列测试（含安全删除）

4. `demo_tailq.c` — TAILQ 双向尾队列**终极测试**（全功能）

所有 Demo 包含：

- 头插 / 尾插 / 指定节点前后插入
- 普通遍历
- **SAFE 安全遍历（遍历时删除专用）**
- 删除节点
- 内存释放
- 判空

## 最重要 API 说明（必看）
### 通用遍历宏
- `XXX_FOREACH`：普通遍历，**遍历时不能删除节点**
- `XXX_FOREACH_SAFE`：安全遍历，**遍历时删除节点必须使用**

### 插入 API
- `XXX_INSERT_HEAD`：头部插入
- `XXX_INSERT_TAIL`：尾部插入（仅 STAILQ / TAILQ 支持）
- `XXX_INSERT_AFTER`：指定节点后插入
- `XXX_INSERT_BEFORE`：指定节点前插入（仅 LIST / TAILQ 支持）

### 删除 API
- `XXX_REMOVE`：删除指定节点
- `STAILQ_REMOVE_CHECK`：带安全检查的删除（防止野指针）

## 使用规范
1. **必须先初始化**：`XXX_INIT(head)`
2. **遍历删除必须用 SAFE 版本**，否则会崩溃
3. **malloc 节点必须手动 free**，库不负责内存管理
4. **优先使用 TAILQ**，功能最全、性能最好、最通用
